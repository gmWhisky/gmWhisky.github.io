求二叉树的最大深度

问题：给定一个二叉树，找出其最大深度。二叉树的深度为根节点到最远叶子节点的距离。

  • 解答：如果二叉树为空，则深度为0；如果不为空，分别求左子树的深度和右子树的深度，去最大的再加1，因为根节点深度是1，要加进去。

求二叉树的最小深度

问题：给定一个二叉树，找出其最小深度。二叉树的最小深度为根节点到最近叶子节点的距离。

解答：

  • 开始的想法是把最大深度的代码直接交换改成最小深度但是测试样例的时候并没有通过,例如[1,2]的时候本应该输出1，结果输出的为2；或者如果出现斜树的情况，也就是说只有左子树，或只有右子树的时候，如果按照刚才说的那个算法来实现的话，得出的答案是0，明显是错误的。
  • 一种就是计算左子树和右子树深度的时候，判断是否等于0，如果等于0，说明该子树不存在，深度赋值为最大值。
  • 第二种就是判断左子树或右子树是否为空，若左子树为空，则返回右子树的深度，反之返回左子树的深度，如果都不为空，则返回左子树和右子树深度的最小值

插入类排序

插入排序

要排序的数组中，假设前n-1（n>=2）个数已经是排好序的，现在要把第n个数插入到前n个已经排好序的数组中，使得这n个数也变成有序的，如此反复循环，使得要排序的数组中的最后一个元素也排好序

每次插入，都从前面的有序子表中查找出待插入元素应该被插入的位置； 
给插入位置腾出空间，将待插入元素复制到表中的插入位置。

  • 直接插入排序

    思路：假设第一个数是排好序的，然后第二个数和第一个数进行比较，如果第二个数比第一个数大，那么说明前两个数排好序，无需做调整，如果第二个数比第一个数小，那么就把第一个数向后移，将第二个数放在第一个数的位置上，抽象出来就是用a[i]和a[i-1]进行比较，如果a[i]>a[i-1]，那么就说明前i个数是已经排好序的，如果a[i]<a[i-1]，就让j=i-1,temp=a[i],然后一边将数据a[j]向后移动，一边向前搜索，直到找到a[j]<a[i]时停止，并将temp放到a[j+1]处即可。

  • 折半插入排序(直接排序的优化)

思路：1.计算 0 ~ i-1 的中间点，用 i 索引处的元素与中间值进行比较，如果 i 索引处的元素大，说明要插入的这个元素应该在中间值和刚加入i索引之间，反之，就是在刚开始的位置 到中间值的位置，这样很简单的完成了折半；
2.在相应的半个范围里面找插入的位置时，不断的用上面步骤缩小范围，不停的折半，范围依次缩小为 1/2 1/4 1/8 …..快速的确定出第i 个元素要插在什么地方；
3.确定位置之后，将整个序列后移，并将元素插入到相应位置。

希尔排序

思路
1 将整个待排元素序列分割成为若干个子序列（有相隔某个怎么增量的元素组成）分别进行直接插入排序
2 然后依次缩减增量再进行排序
3 待整个序列中的序列（增量足够小）时，再对全体元素直接进行一次直接插入排序

例如：将n个记录分成d个子序列
{R[1] , R[1+d] , R[1+2d] , R[1+3d] , … R[1+kd]}
{R[2] , R[2+d] , R[2+2d] , R[2+3d] , … R[2+kd]}
…
{R[d] , R[d+d] , R[d+2d] , R[d+3d] , … R[d+kd]}

d为增量，它的值在排序过程中从大到小逐渐缩小，纸质最后一趟为1

交换类排序

冒泡排序

思路
1： 比较相邻的元素。如果第一个比第二个大，就交换他们两个。
2：对每一对相邻元素作同样的工作，从开始第一对到结尾的最后一对。在这一点，最后的元素应该会是最大的数。
3：针对所有的元素重复以上的步骤，除了最后一个。
4： 持续每次对越来越少的元素重复上面的步骤，直到没有任何一对数字需要比较。

优化

优化一

  • 设置一个标记来标志一趟比较是否发生交换
  • 如果没有发生交换，则数组已经有序

优化二

  • 用一个变量记录下最后一个发生交换的位置，后面没有发生交换的已经有序
  • 所以可以用这个值来作为下一次比较结束的位置

快速排序

1 从序列中随意选择一个数作为基准数（通常是第一个或者最后一个）
2 一趟排序以后，使得无序数列被分割成为左右两个子序列（凡是比基准数大的在基准数右边，凡是比基准数小的在左边）
3 再递归地将左右两个子序列进行快速排序，直至每个子函数只剩下一个元素

桶类排序

桶排序

原理
根据该数组中值的范围，定一个包括所有值的数组，比如一个班的学生的数学分数进行排序那么就确定一个数组int bucket[] = new int [101];这样
就用bucket数组的角标值标识分数，从0到100。然后遍历需要排序的数组，把每个角标所存储的值，作为bucket数组的角标，然后对bucket数组当前角标的值进行++，从而确定了每个分数有多少人得了这个分数，然后再遍历bucket数组，根据bucket 所存储的值的大小去确认当前分数出现的次数，这样就完成了对分数数组进行排序。

基数排序

将所有待比较的数值统一为相同的长度，然后从个位开始，依次进行依次排序，从最低位排序一直到最高为排序完成后，就是有序数列，所需要的时0-9这10个桶，所以基数排序的思想运用到桶排列

选择类排序

选择排序

  • 简单选择排序

    原理
    在一个序列里边找到最小的和第一个数交换，再在剩下地找最小的和第二个数交换 优化：在一个序列里面一次选出两个元素，一个最大，一个最小，最大的放在右边，最小的放在左边，再依次查找

  • 树形选择排序

    首先将n个数据元素两两分组，分别按关键字进行比较，得到n／2个比较的优胜者(关键字小者)，作为第一步比较的结果保留下来， 然后对这n／2个数据元素再两两分组，分别按关键字进行比较，如此重复，直到选出一个关键字最小的数据元素为止。

归并类排序

归并排序

  • 二路归并排序

    思路
    将初始序列的n个记录看成是n个有序的子序列，每个子序列长度为1
    两两归并，得到n/2个长度为2（n为奇数时，最后一个序列长度为1）的有序子序列
    再对长度为2的有序子序列两两归并 得到若干长度为4的子序列
    直到得到一个长度为n的有序序列

  • 自然归并排序

    原理
    自然合并的核心主要是一个Pass函数,这个函数中设置了一个array数组,来存放每一组有序元素的起始元素的下标,最后再将最后一个元素的下标+1存放为array数组的最后一个元素,这样,在后面的合并实现中会显现出这样记录的原因

快慢指针赛跑 (判断是否有环)

原理是：快指针的移动速度是慢指针移动速度的2倍，因此当快指针到达链表尾时，慢指针到达中点。程序还要考虑链表结点个数的奇偶数因素，当快指针移动x次后到达表尾（1+2x），说明链表有奇数个结点，直接返回慢指针指向的数据即可。如果快指针是倒数第二个结点，说明链表结点个数是偶数，这时可以根据“规则”返回上中位数或下中位数或（上中位数+下中位数）的一半。

c
思路：让快指针每次移动2，慢指针每次只移动1 设最后相遇所在的节点为meet
meet存在即有环
从head与meet出发 ，两指针速度一样的时候，相遇时即为环的起点

补充 npm 全安装路径配置
（这是当初安装vscode的时候遇到NodeJS安装问题查的资料）
npm 是模块管理器 用于管理用户require的模板分成全局和本地两种

全局： npm install <你的模块> -g
这样会直接安装在全局路径下，默认在C:\Users\Administator\AppData\Roaming\npm路径下

局部: npm install <包的名字> 将模块安装在dos窗当前指引的路径下

  • 出现 fatal: Could you read from remote repository.的报错

    这表示的是github的权限问题

    解决方法：

    • 先检查是否有SSH keys（我从一开始拿到的这个 后面出现任何错误都没有进行过修改 也不要进行重复获取keys）

    • 然后测试设置是否成功 在git里面输入

    • 我出现的结果是 PTY allocation request failed on channel 0 这表示肯定是配置出现了问题 ： 使用的ptmx设备没有在/dev环境下

      解决方法：直接做一个链表文件，在dev目录下执行 ln -s ./pts/ptmx ./ 或者 直接在 /dev目录下创建一个ptmx的文件就行

  • 解决配置hexo主题过程中将语言换成中文的问题

    开始的时候 只是在站点根目录下修改配置文件_cofig.yml中的language为zh-Hans（简体中文）但是如果我有新增的内容的话，显示还是英文。最后应该是进入到主题目录下的languages文件中修改zh-Hans.yml这样才能显示出菜单项新增的中文内容

  • 有时候修改了主题里面的一些元素之后 刷新没有反应

    解决办法 修改Hexo根目录下_config.yml站点配置目录后 要重新启动Hexo再刷新;修改next主题目录下_config.yml主题配置后，直接刷新就好啦！

  • 网页显示404或者空白网页

    • 404页面是客户端在浏览网页时，服务器无法提供正常的信息或者无法回应
      通俗一点来说就是端口有错，会在页面解析渲染的时候报错页面

      解决方法：先hexo s看一下给出的网址端口是否可以打开，如果不行的话，在git 里面输入“hexo server -p 端口号”来改变端口，并且要在Cltr C之前将网址端口复制到网页中验证。再将本地网站部署（hexo d ，hexo g ）到github，让库生成网页就好啦！！！

    • 如果是空白网页，一般情况下是路径出错 由于github page更新了，过滤掉了source/vendors目录的访问

      解决办法：将source/vendors 修改为 source/lib,然后将将 _internal: vendors修改为_internal:lib 然后修改next底下所有引用source/vendors路径为source/lib。这些地方可以通过文件查找找出来。主要集中在这几个文件中。1. Hexo\themes\next.bowerrc 2. Hexo\themes\next.gitignore 3. Hexo\themes\next.javascript_ignore 4. Hexo\themes\next\bower.json 。修改完毕后，刷新重新g一遍就ok啦。
